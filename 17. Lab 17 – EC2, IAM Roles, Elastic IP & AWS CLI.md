# Lab 17 – EC2, IAM Roles, Elastic IP & AWS CLI

# TASK SET 1 – Guided

## Exercise 1 – Create an IAM Role for EC2

Create an IAM role.

1. Open the **AWS Management Console**.
2. Navigate to **IAM**.
3. Select **Roles**.
4. Choose **Create Role**.
5. Select the **EC2** use case.
6. Attach the following policies:

* `AmazonEC2ReadOnlyAccess`
* `AmazonS3ReadOnlyAccess`

Launch an EC2 instance and attach the IAM role.

1. Navigate to **Amazon EC2**.
2. Select the instance.
3. Choose **Actions → Security → Modify IAM Role**.
4. Attach the newly created role.

Verify the attached IAM role from the EC2 instance.

```bash id="np7gzw"
aws sts get-caller-identity
```

Verify that the returned ARN corresponds to the attached IAM role.

## Exercise 2 – Allocate an Elastic IP

Allocate a new Elastic IP.

```bash id="g0u8vb"
aws ec2 allocate-address --domain vpc
```

Associate the Elastic IP with the EC2 instance.

```bash id="j2mjlwm"
aws ec2 associate-address \
--instance-id i-xxx \
--allocation-id eipalloc-xxx
```

Stop and start the EC2 instance.

Verify that the public IP address remains unchanged.

> **📌 Note:** Elastic IP addresses remain associated with an instance even after it is stopped and started.

# TASK SET 2 – Practice

## Exercise 1 – Create a Launch Template

Create a Launch Template using an existing configuration file.

```bash id="s4vczx"
aws ec2 create-launch-template \
--launch-template-name devops-lt \
--launch-template-data file://lt-config.json
```

Launch two EC2 instances from the Launch Template.

```bash id="up8a3t"
aws ec2 run-instances \
--launch-template LaunchTemplateName=devops-lt \
--count 2
```

Verify that both instances are running.

---

## Exercise 2 – Tag EC2 Instances

Retrieve all EC2 instance IDs.

```bash id="ylmk1w"
aws ec2 describe-instances \
--query "Reservations[].Instances[].InstanceId" \
--output text
```

Tag all instances.

```bash id="wvjnd3"
aws ec2 describe-instances \
--query "Reservations[].Instances[].InstanceId" \
--output text | xargs -I{} \
aws ec2 create-tags \
--resources {} \
--tags Key=Environment,Value=Dev Key=Owner,Value=Vinod
```

Verify the applied tags.

## Exercise 3 – Create an Amazon Machine Image (AMI)

Create an AMI from the running EC2 instance.

```bash id="mv8e99"
aws ec2 create-image \
--instance-id i-xxx \
--name "devops-ami" \
--no-reboot
```

Launch a new EC2 instance using the AMI.

```bash id="cs3w9f"
aws ec2 run-instances \
--image-id ami-xxx \
--instance-type t2.micro \
--key-name devops-key \
--security-group-ids sg-xxx
```

Verify that the new instance launches successfully.

# TASK SET 3 – Challenge

## Exercise 1 – Configure an Auto Scaling Group

Create an Auto Scaling Group.

```bash id="gy92v4"
aws autoscaling create-auto-scaling-group \
--auto-scaling-group-name devops-asg \
--launch-template LaunchTemplateName=devops-lt,Version='$Latest' \
--min-size 2 \
--max-size 5 \
--vpc-zone-identifier subnet-xxx
```

Configure target tracking scaling.

```bash id="xxgj0x"
aws autoscaling put-scaling-policy \
--auto-scaling-group-name devops-asg \
--policy-name cpu-target \
--policy-type TargetTrackingScaling \
--target-tracking-configuration '{
  "TargetValue": 50,
  "PredefinedMetricSpecification": {
    "PredefinedMetricType": "ASGAverageCPUUtilization"
  }
}'
```

Generate CPU load.

```bash id="zwl6lb"
sudo apt install -y stress

stress --cpu 4 --timeout 120
```

Observe the Auto Scaling Group launching additional instances as CPU utilization increases.

## Exercise 3 – Automate Cost Optimization

Create the following Bash script.

```bash id="2uj6zi"
#!/bin/bash

aws ec2 describe-instances \
--filters \
"Name=tag:Environment,Values=Dev" \
"Name=instance-state-name,Values=running" \
--query "Reservations[].Instances[].InstanceId" \
--output text | \
xargs aws ec2 stop-instances --instance-ids
```

Run the script.

Verify that all running EC2 instances tagged with **Environment=Dev** are stopped.

> **💡 Tip:** Automating the shutdown of non-production resources helps reduce unnecessary cloud costs.

# 💡 Best Practices

* Use IAM Roles instead of storing AWS credentials on EC2 instances.
* Allocate Elastic IPs only when required to avoid unnecessary charges.
* Use Launch Templates for consistent EC2 deployments.
* Apply standardized resource tags for governance and cost tracking.
* Regularly create AMIs before major system changes.
* Configure Auto Scaling based on workload demand.
* Automate software installation using User Data.
* Schedule regular cleanup of unused AWS resources.

---

# 🔒 Security Considerations

* Follow the principle of least privilege when assigning IAM permissions.
* Avoid embedding AWS access keys on EC2 instances.
* Restrict Security Group access to required ports only.
* Encrypt EBS volumes containing sensitive data.
* Protect EC2 access using SSH key pairs and security groups.
* Monitor IAM role usage with AWS CloudTrail.
