# Lab 15 – ArgoCD Setup, Auto-Sync & Helm Integration

## 📖 Overview

This lab introduces **ArgoCD**, a GitOps continuous delivery tool for Kubernetes. You will install ArgoCD, connect it to a GitHub repository, deploy applications using GitOps principles, enable automatic synchronization and self-healing, deploy Helm charts through ArgoCD, and implement advanced GitOps patterns such as multi-environment deployments, ArgoCD Image Updater, and the App of Apps pattern.

By completing this lab, you will learn how to:

* Install and configure ArgoCD.
* Access the ArgoCD Web UI.
* Connect GitHub repositories to ArgoCD.
* Deploy Kubernetes applications using GitOps.
* Configure automatic synchronization.
* Enable self-healing deployments.
* Deploy Helm charts using ArgoCD.
* Manage multiple environments with Git branches.
* Automatically update container images.
* Implement the App of Apps deployment pattern.

# TASK SET 1 – Guided

## Exercise 1 – Install ArgoCD

Create the ArgoCD namespace.

```bash id="p6lm8m"
kubectl create namespace argocd
```

Install ArgoCD.

```bash id="r0ww0w"
kubectl apply -n argocd -f \
https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
```

Access the ArgoCD Web UI.

```bash id="yhzj0r"
kubectl port-forward svc/argocd-server -n argocd 8080:443
```

Retrieve the initial administrator password.

```bash id="5fsn7m"
kubectl -n argocd get secret argocd-initial-admin-secret \
-o jsonpath='{.data.password}' | base64 -d
```

Log in using the ArgoCD CLI.

```bash id="pk2zhz"
argocd login localhost:8080 --insecure
```

After logging in, change the default administrator password using the ArgoCD Web UI or CLI.

> **🔒 Security Tip:** Always replace the default administrator password immediately after installation.

---

## Exercise 2 – Connect a GitHub Repository

Add the GitHub repository to ArgoCD.

```bash id="kh3z2o"
argocd repo add https://github.com/<username>/devops-journey.git
```

Alternatively, connect the repository using the ArgoCD Web UI:

1. Navigate to **Settings**.
2. Select **Repositories**.
3. Click **Connect Repo**.
4. Select **HTTPS**.
5. Enter the repository URL.
6. Save the configuration.

> **📝 Note:** Replace `<username>` with the appropriate GitHub username or organization.

---

## Exercise 3 – Deploy an Application

Create a file named **argocd-application.yaml**.

```yaml id="wiy4n6"
apiVersion: argoproj.io/v1alpha1
kind: Application

metadata:
  name: myapp
  namespace: argocd

spec:
  project: default

  source:
    repoURL: https://github.com/<username>/devops-journey.git
    targetRevision: main
    path: k8s/

  destination:
    server: https://kubernetes.default.svc
    namespace: dev

  syncPolicy:
    automated:
      prune: true
      selfHeal: true
```

Apply the manifest.

```bash id="lkv1lz"
kubectl apply -f argocd-application.yaml
```

Open the ArgoCD Web UI and manually synchronize the application.

Verify that the application resources are successfully deployed.

# TASK SET 2 – Practice

## Exercise 1 – Enable Automatic Synchronization

Modify the Kubernetes deployment manifest.

```yaml id="f44x33"
replicas: 5
```

Commit and push the changes.

```bash id="oj8w2y"
git commit -am "scale to 5"

git push
```

Observe ArgoCD automatically detecting the repository change and synchronizing the deployment.

> **📌 Note:** Depending on the refresh interval, synchronization may take a few minutes.

---

## Exercise 2 – Verify Self-Healing

Scale the deployment manually.

```bash id="3c7wgc"
kubectl scale deployment nginx-app -n dev --replicas=1
```

Monitor the deployment.

ArgoCD automatically detects configuration drift and restores the deployment to the desired state defined in Git.

> **💡 Tip:** Self-healing ensures that cluster resources always match the desired Git configuration.

# TASK SET 3 – Challenge

## Exercise 1 – Configure Multi-Environment Deployments

Create separate ArgoCD Applications for development and production environments.

Development environment:

* Git branch: `develop`
* Namespace: `dev`

Production environment:

* Git branch: `main`
* Namespace: `prod`

Use separate values files for each environment where appropriate.

> **💡 Tip:** Managing environments through Git branches enables consistent GitOps workflows across development and production.

---

## Exercise 2 – Configure ArgoCD Image Updater

Install ArgoCD Image Updater.

```bash id="ihjdf4"
kubectl apply -n argocd -f \
https://raw.githubusercontent.com/argoproj-labs/argocd-image-updater/stable/manifests/install.yaml
```

Annotate the ArgoCD Application.

```yaml id="fjntt3"
argocd-image-updater.argoproj.io/image-list: myapp=<account-id>.dkr.ecr.ap-south-1.amazonaws.com/myapp
```

Verify that new container images pushed to Amazon ECR automatically update the Git repository and trigger deployments.

---

## Exercise 3 – Implement the App of Apps Pattern

Create a parent ArgoCD Application that manages multiple child applications.

The parent application should reference the **apps/** directory.

The **apps/** directory contains the Application manifests for each child application.

Example directory structure:

```text id="s1nccq"
apps/
├── app1.yaml
├── app2.yaml
├── app3.yaml
└── app4.yaml
```

Deploy the parent application.

Verify that all child applications are automatically created and synchronized.

> **💡 Tip:** The App of Apps pattern simplifies the management of multiple Kubernetes applications from a single Git repository.

# 💡 Best Practices

* Store the desired cluster state in Git repositories.
* Enable automatic synchronization only after validating deployment configurations.
* Use self-healing to prevent configuration drift.
* Separate development and production deployments using Git branches.
* Use Helm charts for reusable application deployments.
* Protect Git repositories with branch protection rules.
* Organize multiple applications using the App of Apps pattern.
* Monitor ArgoCD synchronization status regularly.

---

# 🔒 Security Considerations

* Change the default ArgoCD administrator password immediately after installation.
* Restrict access to the ArgoCD Web UI using authentication and RBAC.
* Protect Git repositories using branch protection and access controls.
* Store sensitive configuration in Kubernetes Secrets.
* Use least-privilege permissions for ArgoCD service accounts.
* Enable TLS for ArgoCD endpoints in production environments.
