# Lab 18 – Application Load Balancer (ALB)

## 📖 Overview

This lab introduces the **AWS Application Load Balancer (ALB)** for distributing application traffic across multiple Amazon EC2 instances. You will create an internet-facing ALB, configure target groups, implement health checks, enable HTTPS using AWS Certificate Manager (ACM), configure path-based routing, enable access logging, perform blue-green deployments using weighted routing, integrate AWS WAF, and route requests to AWS Lambda functions.

By completing this lab, you will learn how to:

* Deploy an Application Load Balancer.
* Configure Target Groups.
* Register EC2 instances with an ALB.
* Configure health checks.
* Implement path-based routing.
* Enable HTTPS with ACM certificates.
* Redirect HTTP traffic to HTTPS.
* Enable ALB access logging.
* Perform blue-green deployments.
* Protect applications using AWS WAF.
* Route requests to AWS Lambda.

# TASK SET 1 – Guided

## Exercise 1 – Create an Application Load Balancer

Configure the following User Data script on each EC2 instance.

```bash id="d7rm8k"
#!/bin/bash

apt install -y nginx

echo "Server: $(hostname)" > /var/www/html/index.html

nginx
```

Create an Application Load Balancer.

1. Open the **Amazon EC2 Console**.
2. Navigate to **Load Balancers**.
3. Select **Create Load Balancer**.
4. Choose **Application Load Balancer**.
5. Select **Internet-facing**.
6. Select **IPv4**.
7. Choose the appropriate **VPC**.
8. Select the Availability Zones:

   * ap-south-1a
   * ap-south-1b
9. Create an **HTTP (Port 80)** listener.
10. Create a new **Target Group**.
11. Register both EC2 instances with the Target Group.
12. Complete the ALB creation.

Verify that both EC2 instances pass the health checks.

---

## Exercise 2 – Verify Traffic Distribution

Set the ALB DNS name.

```bash id="n9j5r1"
ALB_DNS="your-alb.ap-south-1.elb.amazonaws.com"
```

Send multiple requests.

```bash id="1f6xjw"
for i in {1..10}; do
    curl -s $ALB_DNS | grep Server
done
```

Verify that responses alternate between both EC2 instances.

## Exercise 3 – Verify Health Checks

Stop one EC2 instance.

```bash id="e6l4dw"
aws ec2 stop-instances --instance-ids i-xxx
```

Wait for the ALB health check to mark the instance as unhealthy.

Test the ALB again.

```bash id="xx7zt0"
curl $ALB_DNS
```

Verify that traffic is sent only to the healthy EC2 instance.

# TASK SET 3 – Challenge

## Exercise 1 – Perform a Blue-Green Deployment

Create two Target Groups.

* **blue-tg** (Version 1)
* **green-tg** (Version 2)

Configure weighted routing.

| Blue | Green |
| ---- | ----- |
| 90%  | 10%   |

Gradually shift traffic.

| Stage   | Blue | Green |
| ------- | ---- | ----- |
| Initial | 90%  | 10%   |
| Stage 2 | 80%  | 20%   |
| Stage 3 | 50%  | 50%   |
| Final   | 0%   | 100%  |

Modify the ALB listener rule.

```bash id="4l2y5m"
aws elbv2 modify-rule
```

Verify that traffic gradually shifts to the new application version.

# 💡 Best Practices

* Deploy ALBs across multiple Availability Zones for high availability.
* Configure health checks appropriate to your application.
* Use HTTPS for all production workloads.
* Enable access logging for auditing and troubleshooting.
* Store ALB logs in Amazon S3 with lifecycle policies.
* Use weighted routing for controlled application deployments.
* Protect public applications using AWS WAF managed rules.
* Regularly monitor ALB metrics using Amazon CloudWatch.

---

# 🔒 Security Considerations

* Enforce HTTPS using AWS Certificate Manager.
* Redirect all HTTP traffic to HTTPS.
* Protect applications using AWS WAF.
* Restrict Security Group rules to required ports.
* Enable ALB access logging for security auditing.
* Apply least-privilege IAM permissions for ALB administration.
* Regularly rotate SSL/TLS certificates before expiration.
