# TASK SET 1 — Guided

## Task 1 — SSH into EC2 Instance

Use the following command to securely connect to your EC2 instance.

```bash
chmod 400 key.pem
ssh -i key.pem ubuntu@<EC2-PUBLIC-IP>
```

> [!IMPORTANT]
> Replace `<EC2-PUBLIC-IP>` with the public IP address or DNS name of your EC2 instance.

---

## Task 2 — Explore the Linux File System

List the home directory.

```bash
ls -la /home
```

Read the system user database.

```bash
cat /etc/passwd
```

Check disk usage.

```bash
df -h
```

> [!NOTE]
> The `/etc/passwd` file contains information about all local user accounts.

---

## Task 3 — Create and Read a File

Create a text file.

```bash
echo "Hello DevOps" > ~/test.txt
```

Display the contents.

```bash
cat ~/test.txt
```

# TASK SET 2 — Practice

## Task 1 — Create User **appuser**

Create the user.

```bash
sudo adduser appuser
```

Set the password.

```bash
sudo passwd appuser
```

Add the user to the sudo group.

```bash
sudo usermod -aG sudo appuser
```

Switch to the newly created user.

```bash
su - appuser
```

> [!TIP]
> Verify the user's group membership using:
>
> ```bash
> groups appuser
> ```

---

## Task 2 — Create Application Directory

Create the directory.

```bash
sudo mkdir /opt/app
```

Assign ownership.

```bash
sudo chown appuser:appuser /opt/app
```

Verify permissions.

```bash
ls -la /opt/
```

---

## Task 3 — Find Files Larger Than 10 MB

```bash
find / -size +10M -type f 2>/dev/null
```

> [!NOTE]
> `2>/dev/null` suppresses permission-denied messages.

# TASK SET 3 — Challenge

## Task 1 — Create a System Health Script

Create a Bash script.

```bash
#!/bin/bash

echo "=== System Health ==="

echo "CPU Usage: $(top -bn1 | grep "Cpu(s)" | awk '{print $2"%"}')"

echo "Memory: $(free -m | awk '/Mem/{printf "%.1f%%", $3/$2*100}')"

echo "Disk: $(df -h / | awk 'NR==2{print $5}')"

echo "Top 3 Processes:"

ps aux --sort=-%cpu | head -4 | tail -3
```

> [!TIP]
> Save the script as `system-health.sh` and make it executable:
>
> ```bash
> chmod +x system-health.sh
> ```

---

## Task 2 — Implement Manual Log Rotation

Archive the system log.

```bash
sudo cp /var/log/syslog /var/log/syslog.$(date +%F)
```

Compress the archive.

```bash
sudo gzip /var/log/syslog.$(date +%F)
```

Delete compressed log files older than seven days.

```bash
sudo find /var/log -name "*.gz" -mtime +7 -delete
```

> [!WARNING]
> Verify archived logs before deleting older files.

---

## Task 3 — Harden SSH

Edit the SSH daemon configuration.

```bash
sudo nano /etc/ssh/sshd_config
```

Update the following settings.

```text
PermitRootLogin no
Port 2222
```

Reload the SSH service.

```bash
sudo systemctl reload sshd
```

Reconnect using the new port.

```bash
ssh -i key.pem -p 2222 ubuntu@<EC2-PUBLIC-IP>
```

> [!CAUTION]
> Before closing your existing SSH session, verify that you can successfully reconnect using port **2222**. Otherwise, you may lock yourself out of the server.

# 🛠 Troubleshooting

| Issue                             | Possible Cause                                                | Resolution                                                       |
| --------------------------------- | ------------------------------------------------------------- | ---------------------------------------------------------------- |
| Permission denied                 | Incorrect file permissions                                    | Verify ownership and use `sudo` where required.                  |
| SSH connection refused            | SSH service not reloaded or security group blocking port 2222 | Reload `sshd` and allow TCP port 2222 in the EC2 Security Group. |
| User cannot use sudo              | User not added to sudo group                                  | Run `sudo usermod -aG sudo appuser` and log out/in.              |
| `find` displays permission errors | Searching protected directories                               | Use `2>/dev/null` to suppress permission-denied messages.        |

---

# 💡 Best Practices

* Use least-privilege access for user accounts.
* Regularly monitor system health and resource usage.
* Rotate and archive log files to conserve disk space.
* Disable direct root SSH login to improve security.
* Change the default SSH port only in conjunction with proper firewall and security group rules.
* Verify configuration changes before ending active administrative sessions.
