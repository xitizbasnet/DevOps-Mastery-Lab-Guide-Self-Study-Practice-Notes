# Lab 12 – Helm Charts: Install, Upgrade & Rollback

## 📖 Overview

This lab introduces **Helm**, the package manager for Kubernetes. You will install community Helm charts, customize deployments using value overrides, upgrade and roll back releases, create your own Helm chart, manage chart dependencies, and publish charts to a ChartMuseum repository.

By completing this lab, you will learn how to:

* Install Helm charts from public repositories.
* Manage Helm releases.
* Customize deployments using values.
* Upgrade and roll back Helm releases.
* Create custom Helm charts.
* Manage chart dependencies.
* Publish Helm charts to a ChartMuseum repository.

---

## 🎯 Learning Objectives

After completing this lab, you should be able to:

* Add and manage Helm repositories.

# TASK SET 1 – Guided

## Exercise 1 – Install an Nginx Helm Chart

Add the Bitnami Helm repository.

```bash
helm repo add bitnami https://charts.bitnami.com/bitnami
```

Update the local Helm repository index.

```bash
helm repo update
```

Install the Nginx chart.

```bash
helm install my-nginx bitnami/nginx
```

Verify the installed release and running pods.

```bash
helm list

kubectl get pods
```

> **📌 Note:** Helm installs all Kubernetes resources defined within the chart as a single managed release.

---

## Exercise 2 – View Release Information

Display the status of the Helm release.

```bash
helm status my-nginx
```

Review the output for:

* Release status
* Deployment information
* Service details
* Notes
* Access instructions

---

## Exercise 3 – Uninstall the Helm Release

Remove the Helm release.

```bash
helm uninstall my-nginx
```

Verify that all Kubernetes resources have been removed.

```bash
kubectl get all
```

> **💡 Tip:** Uninstalling a Helm release removes all Kubernetes resources created by that release unless resources have been configured to persist.

# TASK SET 2 – Practice

## Exercise 1 – Install a Chart with Custom Values

Install the Nginx chart with the service type set to **NodePort**.

```bash
helm install my-nginx bitnami/nginx \
--set service.type=NodePort
```

Verify that the service type is configured as **NodePort**.

```bash
kubectl get svc
```

---

## Exercise 2 – Upgrade the Release

Upgrade the existing release by increasing the replica count.

```bash
helm upgrade my-nginx bitnami/nginx \
--set replicaCount=3
```

Verify the deployment.

```bash
kubectl get deployment my-nginx
```

---

## Exercise 3 – Roll Back the Release

Restore the release to revision **1**.

```bash
helm rollback my-nginx 1
```

Verify that the previous replica count has been restored.

```bash
kubectl get deployment my-nginx
```

> **📌 Note:** Helm stores release history, allowing deployments to be rolled back quickly if an upgrade introduces issues.

# TASK SET 3 – Challenge

## Exercise 1 – Create a Custom Helm Chart

Create a new Helm chart.

```bash
helm create myapp
```

Modify the **values.yaml** file to configure your Flask application.

Update the following values:

* `image.repository`
* `image.tag`
* `service.port`

Install the custom chart.

```bash
helm install flask-release ./myapp
```

Verify the deployment.

```bash
kubectl get pods
```

---

## Exercise 2 – Add Chart Dependencies

Update the **Chart.yaml** file.

```yaml
dependencies:
  - name: redis
    version: 17.x.x
    repository: https://charts.bitnami.com/bitnami
```

Download the chart dependencies.

```bash
helm dependency update ./myapp
```

Install the updated chart.

```bash
helm install flask-release ./myapp
```

> **💡 Tip:** Helm automatically downloads and packages chart dependencies before installation.

---

## Exercise 3 – Publish a Chart to ChartMuseum

Install the ChartMuseum Helm plugin.

```bash
helm plugin install https://github.com/chartmuseum/helm-push
```

Publish the chart.

```bash
helm cm-push ./myapp http://chartmuseum:8080
```

Add the ChartMuseum repository.

```bash
helm repo add myrepo http://chartmuseum:8080
```

Install the chart from the repository.

```bash
helm install flask-release myrepo/myapp
```

> **📌 Note:** ChartMuseum provides a private Helm chart repository for publishing and sharing Helm charts across multiple Kubernetes clusters.

# 💡 Best Practices

* Store custom configuration in `values.yaml` instead of modifying chart templates.
* Use `helm upgrade` for application updates instead of reinstalling releases.
* Test chart changes in development environments before production deployment.
* Maintain versioned Helm charts for traceability.
* Use private repositories such as ChartMuseum for internal charts.
* Regularly update chart dependencies.
* Review release history before performing rollbacks.

---

# 🔒 Security Considerations

* Use trusted Helm chart repositories.
* Verify chart sources before installation.
* Scan container images referenced by Helm charts for vulnerabilities.
* Restrict access to private Helm repositories.
* Protect ChartMuseum using authentication and TLS.
* Store sensitive configuration in Kubernetes Secrets rather than `values.yaml`.
