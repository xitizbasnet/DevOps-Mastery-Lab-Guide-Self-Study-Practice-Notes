> **Install Docker, run containers, manage the container lifecycle, expose ports, and inspect running services.**

> [!NOTE]

# TASK SET 1 — Guided

## Task 1 — Install Docker on EC2 and Run Hello World

Install Docker and configure the current user to run Docker commands without using `sudo`.

```bash
sudo apt update && sudo apt install -y docker.io
sudo usermod -aG docker ubuntu && newgrp docker

docker run hello-world
```

> [!IMPORTANT]
> The `hello-world` container verifies that Docker has been installed correctly and is functioning properly.

---

## Task 2 — Run NGINX in Detached Mode on Port 8080

Run an NGINX web server container in detached mode and expose it on port **8080**.

```bash
docker run -d --name web -p 8080:80 nginx
```

Open the following URL in your web browser:

```text
http://<EC2-PUBLIC-IP>:8080
```

> [!TIP]
> Ensure that TCP port **8080** is allowed in your EC2 Security Group before accessing the web server.

---

## Task 3 — View Logs, Access the Container, and Verify NGINX Configuration

View the container logs.

```bash
docker logs web
```

Access the running container.

```bash
docker exec -it web bash
```

List the NGINX configuration directory.

```bash
ls /etc/nginx/
```

> [!NOTE]
> The `docker exec` command allows you to execute commands inside a running container.

# TASK SET 2 — Practice

## Task 1 — Run Three NGINX Containers on Different Ports

Deploy three NGINX containers simultaneously.

```bash
for p in 8081 8082 8083; do
    docker run -d --name web$p -p $p:80 nginx
done

docker ps
```

Verify that all three containers are running.

---

## Task 2 — Inspect a Container and Find Its Internal IP Address

Inspect the container.

```bash
docker inspect web | grep IPAddress
```

Access the container using its internal IP address.

```bash
curl http://<Container-IP>
```

> [!TIP]
> Replace `<Container-IP>` with the IP address obtained from the previous command.

---

## Task 3 — Stop All Running Containers

Stop every running Docker container.

```bash
docker stop $(docker ps -q)
```

> [!WARNING]
> This command stops **all** running containers on the Docker host.

# TASK SET 3 — Challenge

## Task 1 — Run a MySQL Container Using Environment Variables

Deploy a MySQL container.

```bash
docker run -d --name db \
-e MYSQL_ROOT_PASSWORD=root123 \
-e MYSQL_DATABASE=devops \
mysql:8
```

Connect to the MySQL container.

```bash
docker exec -it db mysql -uroot -proot123
```

> [!IMPORTANT]
> Store production database credentials securely using Docker Secrets or a secure secret management solution instead of hardcoding passwords.

---

## Task 2 — Run Redis and Test Connectivity

Create a Docker network.

```bash
docker network create redisnet
```

Deploy a Redis container.

```bash
docker run -d --name redis --network redisnet redis
```

Launch the Redis CLI in the same network.

```bash
docker run -it --network redisnet redis redis-cli -h redis
```

Test Redis.

```text
SET key1 hello
GET key1
```

Expected output:

```text
OK
"hello"
```

> [!NOTE]
> Containers within the same Docker network can communicate using their container names as hostnames.

## Task 3 — Bash Script: Start Containers, Check Health, Report Status, and Clean Up

Create the following Bash script.

```bash
#!/bin/bash

for i in $(seq 1 5); do
    docker run -d --name c$i nginx
done

for i in $(seq 1 5); do
    STATUS=$(docker inspect -f "{{.State.Status}}" c$i)
    echo "c$i: $STATUS"
done

docker stop $(docker ps -q) && docker rm $(docker ps -aq)
```

The script performs the following tasks:

* Starts five NGINX containers.
* Checks the status of each container.
* Displays whether each container is running.
* Stops all running containers.
* Removes all containers from the Docker host.

> [!CAUTION]
> The cleanup command removes **all** Docker containers from the host. Ensure no production containers are running before executing this script.

# 🛠 Troubleshooting

| Issue                                  | Possible Cause                     | Resolution                                                 |
| -------------------------------------- | ---------------------------------- | ---------------------------------------------------------- |
| Docker command not found               | Docker not installed               | Install Docker using `sudo apt install docker.io`.         |
| Permission denied while running Docker | User not in the Docker group       | Run `sudo usermod -aG docker <username>` and log in again. |
| Unable to access NGINX                 | Port 8080 blocked                  | Allow TCP port **8080** in the EC2 Security Group.         |
| MySQL container exits immediately      | Incorrect environment variables    | Verify the required MySQL environment variables.           |
| Redis connection failed                | Containers not on the same network | Verify both containers are attached to `redisnet`.         |
| Container already exists               | Duplicate container name           | Remove the existing container or choose a different name.  |

# 💡 Best Practices

* Use meaningful container names for easier management.
* Keep containers lightweight and focused on a single service.
* Use Docker networks instead of exposing unnecessary ports.
* Store sensitive information securely instead of embedding passwords in commands.
* Regularly inspect container logs to identify issues early.
* Clean up unused containers and images to conserve disk space.
* Verify security group rules before exposing container ports to external users.
