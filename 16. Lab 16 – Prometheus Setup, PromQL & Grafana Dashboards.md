# TASK SET 1 – Guided

## Exercise 1 – Deploy the kube-prometheus-stack

Add the Prometheus Community Helm repository.

```bash id="hnrjgy"
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
```

Update the repository index.

```bash id="cfjlwm"
helm repo update
```

Install the monitoring stack.

```bash id="nd58kv"
helm install prom-stack prometheus-community/kube-prometheus-stack \
--namespace monitoring \
--create-namespace \
--set adminPassword=Admin@123 \
--set prometheus.prometheusSpec.retention=15d
```

Access Grafana.

```bash id="snjicq"
kubectl port-forward -n monitoring svc/prom-stack-grafana 3000:80
```

Login using:

```text id="dxtsqb"
Username: admin
Password: Admin@123
```

Import Grafana Dashboard **1860**.

1. Open **Dashboards**.
2. Select **Import**.
3. Enter Dashboard ID **1860**.
4. Import the dashboard.

> **📌 Note:** Dashboard **1860** is the popular Node Exporter Full dashboard used for monitoring Kubernetes nodes.

# TASK SET 3 – Challenge

## Exercise 1 – Instrument a Flask Application

Modify the Flask application.

```python id="0rmu2i"
from prometheus_client import Counter, generate_latest

REQUESTS = Counter(
    "http_requests_total",
    "Total requests"
)

@app.route("/metrics")
def metrics():
    return generate_latest()
```

Create a ServiceMonitor.

```yaml id="5z3w0l"
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor

metadata:
  name: flask-monitor

spec:
  selector:
    matchLabels:
      app: flask

  endpoints:
    - port: http
      path: /metrics
```

Apply the ServiceMonitor and verify that Prometheus begins scraping application metrics.

## Exercise 3 – Build an SLI/SLO Dashboard

Create dashboard panels using the following PromQL queries.

### Availability

```promql id="mf2bvt"
(1 - rate(http_requests_total{status=~"5.."}[5m]) /
rate(http_requests_total[5m])) * 100
```

### P95 Latency

```promql id="v20qpx"
histogram_quantile(
  0.95,
  rate(http_request_duration_seconds_bucket[5m])
)
```

### Error Rate

```promql id="shxej6"
rate(http_requests_total{status=~"5.."}[5m])
```

> **💡 Tip:** SLI (Service Level Indicator) metrics are commonly used to measure application reliability, latency, and availability.

# 💡 Best Practices

* Deploy monitoring in a dedicated namespace.
* Secure Grafana administrator credentials.
* Organize dashboards by application or environment.
* Configure meaningful alert thresholds to reduce alert fatigue.
* Instrument applications with custom metrics where appropriate.
* Use ServiceMonitors for automatic target discovery.
* Monitor application health in addition to infrastructure metrics.
* Regularly review and optimize PromQL queries.

---

# 🔒 Security Considerations

* Restrict access to Prometheus and Grafana using authentication.
* Secure Alertmanager webhook endpoints.
* Protect SMTP credentials and Slack webhooks using Kubernetes Secrets.
* Limit access to monitoring dashboards using RBAC.
* Enable HTTPS for Grafana and Prometheus in production.
* Regularly audit monitoring and alerting configurations.
